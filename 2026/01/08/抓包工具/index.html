Charles Vs BurpSuite

Charles

不用在浏览器里边设置代理，可以直接使用

所有代理与 HTTPS 捕获操作均通过 Charles 完成配置

前提步骤：安装证书

  1. 打开 Charles，进入菜单栏：
    Help → SSL Proxying → Install Charles Root Certificate
  2. 系统会打开证书安装窗口，将证书安装到“受信任的根证书颁发机构”存储中（Windows）或钥匙串中设置为“始终信任”（macOS）。

开启 SSL 代理

  1. 进入代理设置：
    Proxy → SSL Proxying Settings

  2. 在 SSL Proxying 标签页中：

    • ✅ 勾选 Enable SSL Proxying

  3. 在 Proxy 菜单中确保也勾选：

    • ✅ Windows Proxy（Windows 系统代理，方便浏览器直接走 Charles）

      可尝试在 SSL Proxying Settings 的 Locations 中添加对应域名与端口（如 *:* 表示捕获所有 HTTPS 流量）。

      c

使用 BurpSuite 需要在浏览器中手动设置代理，且每次使用前需开启电脑设置里边的代理。

部分应用（如微信小程序）可能需要额外配置或配合其他工具。

提前步骤：下载并导入证书（中文版）

1.1从 BurpSuite 导出证书

  1. 打开 BurpSuite，进入：
    代理 → 代理设置 → 导入/导出 CA 证书
  2. 选勾选“DER 格式证书”，保存为 burp.der（也可选择 PEM 格式，某些场景兼容性更好）。

1.2 浏览器导入证书

  • Edge：
    1. 浏览器设置 → 证书 → 管理证书 → 管理从Windows导入的证书 → 导入
    2. 选择 受信任的根证书颁发机构 → 导入
    3. 选择之前导出的 burp.der，（导入时选择“所有文件”)
    4. 同样的步骤，选择 中间证书颁发机构再操作一遍

注：建议将证书同时导入系统证书存储（Windows 证书管理器或 macOS 钥匙串），以便部分客户端应用也能信任 Burp 的 CA。

步骤 2：浏览器代理设置

2.1 手动配置浏览器代理

  1. 打开浏览器设置

  2. 浏览器设置 → 代理 → 打开代理设置 → 手动设置代理

  3. 跳转到电脑的设置页面里：

    • 地址：填写本机 IP（可通过 ipconfig / ifconfig 查看，通常为 192.168.x.x 或 127.0.0.1）

    • 端口：Burp 默认监听 8080

    • 下面那个通常不填，否则有些场景下无法使用

特殊场景：抓取微信小程序包

1 .可以直接在微信中设置代理（类似于在浏览器里设置代理）

但是！！！

  • 部分小程序会检测代理并拒绝连接
  • 微信内置浏览器可能忽略系统代理
  • 而且可能会导致微信部分功能不能使用

2. 解决方案：Charles + BurpSuite 协同抓包

  1. Charles 转代理：

    • 打开 Charles：Proxy → External Proxy Settings

    • 勾选 Use external proxy servers

    • 将 HTTP 与 HTTPS 代理指向本地的地址 127.0.0.1:8080（即 BurpSuite 监听地址）

      • BurpSuite 监听地址

        打开BurpSuite：代理 → 代理设置→ 代理监听器

  2. 流量走向：

    小程序 → Charles（端口 8888） → BurpSuite（端口 8080） → 目标服务器

补充说明

  • Burp 中的 Intercept 开关用于控制是否拦截请求，不影响代理运行

  • 可以过滤无关流量

    • 在 Burp Proxy → Options 中的 Proxy Listeners 可设置绑定特定 IP

    • 在 Scope 标签中可定义目标范围，避免捕获过多无关请求

    使用建议：开始抓包前，先关闭浏览器不必要的插件，并清除缓存，避免干扰。抓取移动端流量时，建议先在同一 Wi-Fi 下测试连通性。

Proxifier 转代理及抓包相关笔记

一、监听小程序工具协同使用

  1. 仅用Charles：电脑代理关闭时，可直接用Charles监听小程序流量。
  2. BurpSuite+Charles：使用BurpSuite（BP）需打开电脑代理，再通过Charles转发流量。
  3. Proxifier+其他工具：用Proxifier转代理时，电脑代理可关闭，无需手动开启。

二、Proxifier 配置步骤

  1. 添加代理服务器
    操作路径：Profile → Proxy servers，添加地址 127.0.0.1，端口 8080（对应HTTPS协议）。
  2. 设置代理规则
    操作路径：Profile → Proxification Rules，针对进程名（如 weixin*.exe）配置规则，指定使用上述 127.0.0.1:8080 的HTTPS代理。
  3. 进程名获取方式
    • 从任务管理器中查找目标程序进程名；
    • 或通过预览功能快速定位。

总结：

一、抓包限制与适用场景

  1. 协议限制：仅能抓取 HTTP/HTTPS 协议的流量，其他协议无法捕获。
  2. 适用对象：可抓取浏览器、APP、小程序、PC应用/客户端的流量。

二、抓包的核心用途

  1. 定位包含关键信息的数据包，用于安全测试；
  2. 分析数据包中的参数，检测信息泄露等安全问题；
  3. 挖掘数据包中的业务逻辑参数，辅助理解系统交互机制。
